# Как рассказывать про TicketFlow на собеседовании

## Короткий питч

TicketFlow - это сервис бронирования билетов, рассчитанный на конкурентные заказы. Пользователь регистрируется, видит события, создает заказ, а система атомарно резервирует билет, публикует событие в Kafka и асинхронно создает уведомление.

## Почему тут микросервисы

- `user-service` отвечает только за пользователей и токены.
- `event-service` владеет каталогом событий и остатками билетов.
- `order-service` принимает заказы и координирует сценарий покупки.
- `notification-service` подписывается на события заказов и не блокирует основной путь покупки.
- `api-gateway` дает один публичный HTTP-вход.

## Как решается конкурентность

Главная защита от overselling находится в PostgreSQL:

```sql
update events
set available = available - $1
where id = $2 and available >= $1
returning available;
```

Если 100 пользователей одновременно покупают последние 20 билетов, PostgreSQL сериализует изменения одной строки. Успешными станут только те запросы, которые успели выполнить условие `available >= quantity`.

## Зачем Redis

Redis используется в двух местах:

- кэш списка событий в `event-service`, чтобы не ходить в PostgreSQL на каждый `GET /events`;
- идемпотентность заказов в `order-service` через `SETNX`, чтобы повторный клиентский retry с тем же ключом не создал второй заказ.

## Зачем Kafka

`order-service` публикует событие `order.created` или `order.rejected`. `notification-service` читает эти события и создает уведомления. Так уведомления не замедляют создание заказа, а новые подписчики можно добавить без изменения `order-service`.

## Что можно улучшить

- Добавить transactional outbox: сейчас между записью заказа в БД и публикацией в Kafka теоретически возможна потеря события.
- Заменить учебный password hash на bcrypt или argon2.
- Добавить refresh tokens и нормальную ротацию секретов.
- Добавить OpenTelemetry traces, structured logs и Prometheus metrics.
- Добавить retry с backoff и dead-letter topic для Kafka consumer.
- Разнести таблицы по отдельным базам или схемам, если хочется строже показать владение данными.
- Добавить интеграционные тесты через testcontainers.
